Closed Lab 1

(n × 3) + 6

  • Overview
  • Procedure
  • Requirements
  • Submission

Overview

Practice using Logisim and build the F(n) = n × 3 + 6 circuit in simulation. Refer to the Logisim Overview.

Pop Quiz: How many output bits will you need to represent f(n), if n can be any 2-bit number?

If you have correctly answered the above question, then you will be able to make a truth table outlining all possible inputs to your machine and their corresponding outputs. This will be your first step. After that, you'll be able to analyze your truth table and determine the Combinational Logic which correctly generates each output bit based on the two input bits. You will want to diagram this logic with pencil and paper. Based on that diagram, you will finally be able to design the circuit itself.

Procedure

  1. Refer the Logisim Overview and be sure to you have created the indicated generic.circ file.
  2. Refer to and familiarize yourself with the Logisim Conventions. Especially make sure that you follow the conventions regarding pin direction.
  3. Draw the truth table that represents the behavior of this circuit — show all possible inputs and correct outputs for each input
  4. Analyze the table and determine the combinational logic that guides each output bit. You should develop an algebraic expression for each output bit.
  5. Diagram the combinational circuit with a complete logic design diagram.
  6. Open Logisim and open your generic.circ file.
  7. Now, use Save As and save the circuit as initial_labs.circ
  8. Look at the top of the libraries tree and right click on the main circuit and choose Rename Circuit.
  9. Rename the main circuit - "x3plus6"
  10. Add the following I/O:
    • 2 1-bit inputs facing east each with appropriate label
    • 4 1-bit outputs facing west each with appropriate label
  11. Use Logisim to create your first closed lab
  12. Save the circuit.
  13. Read the following requirements before submitting your circuit for grading.

Requirements

  • As always, all requirements from the lab syllabus apply, as well as the following requirements
  • The circuit must be interactive. Toggling the input pins with the Poke Tool should cause the output value (represented by the four output pins) to update instantly and correctly.
  • The circuit must be aesthetically pleasing. Arrange inputs, outputs, and other devices so that wires flow in parallel and remain untangled. One should be able to follow the flow of the logic by the flow of the wires.
  • Basic conventions of an aesthetically pleasing circuit:
    • Left to Right flow. Inputs on the left, outputs on the right, and a general left-to-right logical progression.
    • Organization: How the circuit is laid out should reflect and guide how one is supposed to think about the circuit.

Submission

  • A zipped file containing:
    • initial_labs.circ
