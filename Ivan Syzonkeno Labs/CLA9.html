Overview

As you are by now beginning to realize, nothing happens in a sequential circuit until the clock ticks, and what happens on a given tick of the clock is determined entirely by the state of that circuit when the tick occurs. So if the system is in state A when the clock ticks, it goes to state B.

We control our computer by controlling its state. The word "state", by the way, simply refers to the current configurations of everything in the machine which can change. In our machine, the only things that change are flip-flops and wires. So what the computer is going to do on any given clock tick is solely determined by which wires are hot when the clock wire goes hot.

Every device you've created so far has one or both kinds of inputs: data and control. We generally put the data inputs on the left and the control inputs on the bottom of the part, to separate them. The data inputs are simply bundles of wires which, by being high or low, serve to represent the information we are trying to process. The control inputs are single wires that "control" or determine the behavior of each component. For example, we will need control wires indicating when to count, when to clear, when to load data, etc. The control inputs are generated by the Control Unit of the computer. The CU produces all the control signals which, in combination with data provided by the user, make up the computer's current state and thus determine what the computer will do next.

In previous labs you learned that the Control Unit for the RSC requires two major components in order to do its job:

  1. A D Signal Generator which decodes an instruction and sends out D signals (d0, d1, d2, ..., d15). This tells our computer WHAT microcode action to do.
  2. A T Signal Generator which sends timing signals (t0, t1, t2 ..., t7) indicating the current step in the machine cycle to be performed. This indicates WHEN the computer does that piece of microcode.

In this lab, you will begin creating the combinational circuits that use the D and T signals to control the PC, DR, and AR registers. Also, in this lab, you will learn how to create complicated combinational circuits "easily" using Logisim. In the real world, complicated combinational circuits can be created using different computer-aided design tools such as PLDs (programmable logic device) which facilitates the actual hardware fabrication of the circuit.

We need to make very sure that all registers/counters in the RSC load ONLY at the proper time. Therefore, in this lab and all future labs related to the Control Unit, make sure that the registers/counters load ONLY on the upswing of the clock. This means that load signals need to be on before the clock goes high.

NOTE: All PLDs will be generated by the Combinational Analysis Tool. Whatever circuit this Tool generates, do not re-orient the input or output pins. This is one time when some circuits may go against convention.

Procedure

PC Bus Signal PLD

First we will go over how to build a typical 'Bus' signal PLD.

  1. Examine the microcode for the RSC. Look through it for every reference to the PC and make a list of the operations it has to perform. You will learn that to fully control the PC register, you will have to provide its Load, Count, and Bus signals. We will begin with the Bus signal since that is the easiest.
  2. Look specifically for instances in which the PC sends its contents to another register. You will find only two: on t0 and t2. Now, reduce this information to a logical expression: PC_BUS = t0 + t2. Next you should convert this into a language which Logisim understands. In Logisim negation is expressed with a preceding exclamation point, an AND operation is assumed wherever two terms are separated by a space and nothing else, and an OR is expressed with a '+' sign.
  3. Open Notepad (or your favorite editor) and create a new text file called Control_PLDs.txt to store your list of boolean expressions
  4. Record your Logisim capable boolean equation for PC_BUS in the text file and save the file.
  5. Open a new, blank circuit file in Logisim. Save it as PLDs.circ and rename its main circuit to test. You may use this circuit to test the PLDs you are going to create.
  6. Now, through the Window menu, open Combinational Analysis. You've used this tool before, to create a PLD from a table. Today, we will use the Expression-Analysis system. Enter the appropriate inputs for the PC_BUS PLD on the Inputs tab. These should be t0 and t2.
    Inputs should always be in ascending order with t's on top, clock in the middle, d's next, and z on bottom (if an input is not needed, do not include it in your Inputs list). This bolded text is absolutely crucial in helping you avoid confusion later on and helping expedite the grading process. Please ensure that you do this properly. Feel free to ask me if you're aren't sure about how you're doing it.
    For example, AR_LD inputs would be arranged:
      • t0
      • t2
      • t4
      • clock
      • d1
      • d2
    You can achieve this by the order in which you add your inputs. If you create the circuit and discover that the inputs are out of order, you can simply, and easily, rearrange them on the breadboard with minimal rewiring.
  7. Enter the appropriate output on the Outputs tab: PC_BUS. Proceed to the Expression tab. Select PC_BUS from the output drop-down list. Next, enter your expression in the space below. Copy in everything to the right of the "=" from your text file. For instance:
    t0 + t2
  8. Once the expression is in place, you must click the ENTER button to register the expression to the chosen output. Otherwise, the following step will only create a blank part. Once the expression is registered, click Build Circuit. Name the circuit after its output, PC_BUS, and generate. Be sure to select the PLDs.circ as the Destination Project.

AR Load Signal PLD

The formulas for the Load signals are a slightly more difficult, but have the same basic idea.

  1. Now, build a PLD for the AR_LD signal. You should first examine the microcode and determine when the control signal for the AR_LD signal should be ON. After examination of the microcode, your expression should appear as:

    AR_LD = t0 !clock + t2 !clock + t4(d1 + d2) !clock
    or
    AR_LD = (t0 + t2 + t4(d1 + d2)) !clock

    Remember that the !clock has been added above to make sure that the AR loads only on the upswing of the clock. Make sure that you understand how this expression was generated. ONLY synchronous operations (expressions with LD or INC) need a !clock term!
  2. Use whichever version you choose, record it in your text file and save the addendum.
  3. Use the same procedure which created the PC_BUS PLD to create a PLD for AR_LD. Enter inputs, enter outputs, register expressions, and build the circuit. Name it after its output or outputs (in this case, AR_LD).

More PLDs

Continue building the following PLDs the same way you built the two previous PLDs.

Regarding SC_CLR, which clears the Sequence Counter: this is how you set the machine back to t0 after completing those operations which take less than 8 clock cycles to complete. (A quick review of the microcode will tell you that this includes all the operations there are.) SC_CLR should activate on the next t value after the last micro-operation is performed for the currently executing operation. For example, if the last step in a JMP operation occurs on t5, then t6 in combination with the appropriate D signal, should activate SC_CLR, which will instantly reset the counter to zero. Lest this circuit end up requiring every single D line, I encourage you to consider the problem in terms of a default case and non-conforming cases: identify on which t value the largest number of operations will conclude, and treat it as a default case. Establish an expression for all other cases and then modify it to use the default condition if none of the others apply.

One other special consideration: the Z register is unique among all the registers. First of all, you will notice that when the Z register loads a value, that value depends on the contents of the Accumulator. You do NOT need to worry about this in your PLDs. Just command the Z register to load on the appropriate D and T combinations. Later, when you build its encapsulating circuitry, you will make sure that it always loads the correct value.

You will also notice in the microcode that several control signals for other registers do sometimes depend on the value currently stored in Z. This you DO have to address in your PLDs. Simply add another input called Z and incorporate it into your logical expression.
