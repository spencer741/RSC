Overview

The RSC, like any computer, can only be called a computer if it can process and remember information. Processing will be addressed when we finish the ALU. The ALU by itself won't do any good if we can't remember the answer and build on it. Thus, an ALU is always sandwiched in some fashion between registers â€” registers to provide input and registers to capture output. Today, you will begin construction of those registers, specifically the simplest sort: a one-bit register consisting of a single flip-flop.

Registers have many capabilities. However, we need a 1-bit register that is able to perform the following operations:

  1. It should be able to load data on demand. If the load signal is active, then on the next clock tick, it should store the value present on its data line (0 or 1).
  2. It should be able to clear data on demand. When the clear signal is active, it should immediately flip to zero (without waiting for the next clock tick).
  3. It should produce a 1-bit output called data_out, which simply connects to the flip-flop's Q output.

Therefore, you should have a total of four 1-bit inputs for this circuit (be sure they face the appropriate direction according to our conventions!):

  • data
  • load
  • clear
  • clock

A clock signal is necessary to synchronize the events of the register with other components of the RSC. This register, along with many other components of the RSC, would do us no good if we were unable to coordinate or synchronize when a certain component does a certain task. We will achieve this coordination by having one master clock for the entire RSC, which will be tied into all components needing a clock. The 1-bit register is one of those devices. This is the reason we need an input pin for the clock signal and not an actual clock from the Logisim library. If you ever tried it out, using a Clock component on a circuit doesn't provide a pin on the chip when you drag it into another circuit. Using the actual 1-bit input labeled "Clock" allows the master RSC clock we'll add later to come into our 1-bit Register and control precisely when things happen.

A Few Hints:

This circuit's construction is very important since we will be using it in a later lab. The circuit will be designed around a JK flip-flop. It is possible to make a one bit register with other types of flip-flops, but for our purposes, JK flip-flops work very well. The JK flip-flop in Logisim works the same way you were taught a JK flip-flop works in lecture, but with one addition. The JK flip-flop in Logisim has an enable pin located on the bottom part of the chip in the middle. The enable pin does exactly what it sounds like, it enables the chip to work. If the enable pin has an input of 0, then it does not matter what values are on the J and K inputs, the flip-flop will not work. Keep this in mind when designing your circuit. You will need to create a state table for the circuit, which will look like the following (see below for more details):

LoadDataQ(t)Q(t+1)JK
           
           
           
           
           
           
           
           

Load and Data are the inputs to your circuit. Q(t) is the output of your circuit and J and K are the inputs to the JK flip-flop you will be using as a base for the circuit. However, when designing your state table, Load, Data, and Q(t) should all be treated as inputs (label them A, B, and C, respectively, in your table if you'd like), as the value of Q(t+1) relies on the current value of Q(t). Also, if you recall, we mentioned earlier that you would be needing a Clear input as well. If the Clear input is going to be part of our circuit, why didn't we include it in the state table? The answer is that the Clear signal of the register is dealt with asyncronously. Asynchronous simply means do it without needing to sync with anything, hence do the action when prompted without care for the clock or any other input. We could have included the Clear input in the state table, but it would have no effect on the outcome. The Clear input will come into play later. Keep in mind that when Clear is 1, the output is 0 no matter what. After your state table is complete, you should use K-maps to determine the appropriate circuits needed to feed the inputs of the J and K inputs on the flip-flop.

In case you hadn't noticed before, the JK flip-flop in Logisim has two asynchronous input pins, located at the bottom of the flip-flop (labeled with a 1 and a 0). Both of these pins operate without regard to the clock signal and the enable signal. The input at the south-east portion of the flip-flop (the one labeled 0) is an asynchronous reset pin. When this input = 1, the output of the flip-flop is set to 0 and will remain for as long as the input = 1. When this input = 0, the flip-flop will operate as expected. The input at the south-west portion of the flip-flop (the one labeled 1) is an asynchronous set pin. When this input = 1, the flip-flop will be set (go to 1). When this input = 0, the flip-flip will operate as expected. When creating your circuit, make sure you are using (or not using) these two inputs correctly.

Once you have completed this device, test it thoroughly to see that it loads 1s and 0s on command, that the clear signal asynchronously clears the register, and then (while active) prevents any attempt to load a 1 (remember the note above about the enable pin?). Clear should supersede everything. Log your tests, and make sure that the log you submit demonstrates conclusively the full function of the register you have created.

Procedure

  1. Read the Overview carefully, then review the Requirements for the 1-bit register.
  2. Design the register, using a JK flip-flop as its core. Draft state table as necessary to ensure that it meets Requirements.
  3. Open Logisim and open the generic.circ.
  4. Use SAVE AS and save the circuit as custom_reg.circ. (NOTE: If you have already created a custom_reg.circ file, then just open your existing one. You do not want to overwrite your custom registers file with a blank circuit.)
  5. Rename the main circuit in the project library to 1bit_reg (Select circuit and change name in properties)
  6. Add the following inputs and outputs:
    • One 1-bit input facing east labeled Data
    • One 1-bit input facing south labeled Clock
    • Two 1-bit inputs facing north labeled Load and Clear
    • One 1-bit output facing west labeled Data_out
  7. Add a JK flip-flop from the Memory folder as well as any combinational logic you will need, per your design.
  8. Wire together all components together appropriately.
  9. Test thoroughly. See the bottom of this page for an example log table for this assignment. For grading purposes, create a log file called 1bit_regtestlog.txt.
  10. Submit to elearn under appropriate drop box.

    • All the usual, plus: This circuit should be built from the ground up. The only prefabricated non-standard part you may use is the JK flip-flop. Other than that, it should be all wires and gates.

      • When Load is active, load the value of the Data line on the next up-tick of the clock.
      • When Clear activates, immediately clear content to zero and as long as Clear is active, prevent any other signal from commanding the flip-flop to up-swing.
    • The final part should have 4 inputs: Data, Load, Clear, and Clock.
