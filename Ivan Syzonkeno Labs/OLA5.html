RSC Registers

  • Overview
  • Procedure
  • Requirements
  • Submission

Overview

In this lab you will build the 32-bit registers of the RSC system. These 32-bit registers in the system each have their own functional needs. Here's a list of the registers and their functionality for the RSC.

Table 1

Register
Functionality
ACC Load, Output, Bus Out, Clear
AR Load, Output
DR Load, Output, Bus Out
IR Load, Output
OUTR Load, Output
PC Load, Output, Bus Out, Increment
R Load, Output, Bus Out

As noted in the above table, there are four, functionally different types of registers in the list. Therefore, in this lab you will create each of these different types of generic registers and lastly, create the above registers using these generic registers as subcircuits. This lab is divided into phases. You should follow the instructions in each phase, making sure each phase is correct before proceeding to the next phase.

Table 2

Phase Purpose
0 Add circuits to the previous custom_reg.circ project
1 Create generic register that can LOAD and OUTPUT
2 Create generic register that can LOAD, OUTPUT and produce a BUS OUTPUT
3 Create generic register that can LOAD, OUTPUT, produce BUS OUTPUT, and CLEAR on demand
4 Create generic register that can LOAD, OUTPUT, produce BUS OUTPUT, and COUNT
5 Create the ACC, AR, DR, IR, OUTR, PC, and the R registers

Procedure

Before we start with the procedure, keep in mind the Logisim Conventions we have set forth in this class. They are very important in this lab.

Phase 0

  1. Open Logisim and save the blank file as custom_reg.circ (or open your custom_reg.circ file if it was created in an earlier lab). If there is still a circuit labeled main, make sure it's empty and delete it. Click here to get the custom_reg with the 1bit_reg circuit. Then add the following circuits to this project:
    • Register
    • BusRegister
    • ClearRegister
    • CountRegister
    • ACC
    • AR
    • DR
    • IR
    • OUTR
    • PC
    • R

Phase 1: The Register
In this phase we'll build the the simplest and most straight forward register. This register can LOAD on demand and produce an OUTPUT. Below is the procedure to produce this register. We'll be using Logisim's built-in register component to simplify grading and to produce a better simulation of the system later on. Logisim's register has 5 input/output pins: Data In, Enable, Clock, Clear, and Data. This register loads when the enable pin is 1 and the clock goes from 0 to 1. This register should NEVER clear.

  1. Add the following I/O:
    • One 32-bit input labeled Data_In
    • One 32-bit output labeled Data_Out
    • Two 1-bit inputs labeled Clock and Load
  2. Add one 32-bit Register from the Memory Folder in the library list
  3. Wire the circuit according to the requirements stated above
  4. Log the circuit. The log file name will be Register_log.txt. Show that the circuit syncronously loads when told.

Phase 2: The Bus Register
Several registers in the RSC will be connected to a bus system. Therefore, these registers must have an additional output (a tri-state output). We will create this generic type of register in this phase which is a pretty simple progression from Phase 1. You'll be taking the previous circuit and adding in a second output system.

  1. Open the BusRegister circuit.
  2. Create a circuit that is identical to the Register circuit, with the addition of the following I/O:
    • One 32-bit output labeled Bus_Out (should be placed below Data_Out)
    • One 1-bit input labeled Bus
  3. Wire the new I/O using a Tri-State Buffer, so that the output of the register is sent to the Bus_Out when Bus is 1
  4. Log the circuit. The log file name will be BusRegister_log.txt. You only need to show that Bus_Out produces an output ONLY when the Bus signal is 1.

Phase 3: Clear Register
Now, we need a register that can load, have two outputs (Data_Out and Bus_Out), and can clear on demand.

  1. Open the ClearRegister circuit.
  2. Create a circuit that is identical to the BusRegister circuit, with the addition of the following I/O:
    • One 1-bit input labeled Clear
  3. This circuit should work identically to the BusRegister circuit, with the exception of being able to clear on demand. Wire the circuit so when the Clear input is 1, the Data_Out will clear.
  4. Log the circuit. The log file name will be ClearRegister_log.txt. You only need to show that the Data_Out will clear when the Clear input is 1.

Phase 4: Count Register
This phase will be a little more difficult. You'll be adding functionality to the register that is more invasive than in previous phases. We'll be adding the ability to increment, or count, in this register. You'll be using some additional components to make this happen.

  1. Open the CountRegister circuit.
  2. Create a circuit that is identical to the BusRegistercircuit, with the addition of the following I/O:
    • One 1-bit input labeled Count
      This will cause the register to increment on the next clock cycle. This signal has priority over Load. Note: This is not a one-shot event, for as long as Count is high (1), the register's value will increment with each clock tick.
  3. Since this register needs the ability to increment, you will also need to bring in Logisim's 32-bit adder into this circuit. This adder should be set-up so that it adds 1 to the data contained in the register.
  4. This circuit should only count on demand. Therefore, you should note that if Count = 0 and Load = 1, the register should load data contained in the Data_In input. If Count = 1 and Load = 0, the register should load data coming from the adder. Therefore, you will need to add a 32-bit 2x1 multiplexer to this circuit which will choose the appropriate data to be loaded into the register. Note: Count has priority over Load. That is, if Count = 1 and Load = 1, the register should only increment on the next clock cycle.
  5. Log the circuit. The log file name will be CountBusRegister_log.txt. Use the following data to test the circuit.
    DATA (in hex) COUNT LOAD BUS BUS OUTPUT OUTPUT
    00000005 0 0 0 _______ _______
    00000005 0 1 0 _______ _______
    00000005 1 0 0 _______ _______
    00000024 1 1 1 _______ _______
    00000017 0 1 1 _______ _______

Phase 5: Encapsulation
Now, we need to create the RSC's 32-bit registers. Looking at the list of registers in Table 1 above, you should match registers with their appropriate functionality. Use the following guidelines for each register.

  1. Open a register circuit. (AR for example)
  2. Add an appropriate sub-component for this register. For example, when creating the AR register, you should bring in your Register as a sub-component since the AR only needs a Load, Input, Output, and Clock signal.
  3. Bring in and label all needed I/O pins.

Requirements

  • All requirements from the syllabus and 3130 Logisim Conventions.
  • Any required functionality listed above.
  • Set the Shared Label for all the non-generic register circuits.
