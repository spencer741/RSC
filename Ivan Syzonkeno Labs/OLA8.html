Programming the RSC

Overview

In order to write programs for the RSC, you must first learn the assembly language for this computer. Your microcode listing contains all the assembly language instructions. Use these instructions to write two different programs as described below. You will be using an RSC emulator to write and test your program. Download the RSC emulator from the class page and read the READ ME FIRST!.txt file on how to run the emulator. Although we will be using the assembly language from our microcode, the emulator uses slightly different spellings of some of the instructions.  Use the emulator's version of each instruction when writing your programs.  If you don't use the same spelling, you program will not compile properly.

Procedure

  1. Write a program which will subtract 8 from 10 (decimal) and display the output. Use the emulator to compile and run the program to verify that it works. Once you have a working program, create a text file called ram1.txt in notepad containing the Assembly Code.
  2. Write a program which will perform the following operations, store each result in a separate memory address and display each value. This program must use every instruction available, with the exception of jump and jumpz. These two are not required, but you may use them if you like. Save the Assembly code in notepad as ram2.txt: (The following values are in Hex)
    • A9 + 2B
    • A9 - 2B
    • A9 AND 2B
    • A9 OR 2B
    • ASHR A9
    • AF + 1
    • NOT D1
  3. Come up with a program of your choosing which solves a common problem. The program must contain a loop(!) and produce some output. Save the Assembly code in notepad as ram3.txt. Fully comment your code along with a description of what it does! Your program idea must first be presented and accepted by the lab instructor. You may either email your proposal or come by during office hours to present your proposed project. Until you have confirmation from the lab instructor, you must assume that your program will not be accepted.

    If you need an help coming up with an program idea take a look here: Fall 2013 or Spring 2013

    You may turn in your ram3.txt program with this lab or wait and turn it in when you submit your final RSC project. If you decide to wait, you will not receive a grade on this lab until your final RSC has been demonstrated with the ram3 program you write. However, you should present your idea within a week from lab assignment.

Requirements

  • Text files must be written in the emulator version of RSC Assembly code
  • Your files must work properly using the Emulator and perform desired operations. Therefore, make sure that your ram3.txt file includes comments so that the lab grader will know the expected output.

Submission

  • A zipped file containing:
    • ram1.txt
    • ram2.txt
