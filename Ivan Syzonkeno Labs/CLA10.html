Closed Lab 10

Control Signals #2

  • Overview
  • Procedure
  • Requirements
  • Submission

Overview

In this lab you will build 7 PLD circuits. You will use the same method discussed in Closed Lab 9 to create the following circuits. Note that each PLD should be named as indicated:

  • R_LD
  • R_BUS
  • S_LD
  • ACC_CLR
  • ACC_LD
  • ACC_BUS
  • ALU_SEL
    • A note about the ALU_SEL PLD: remember that the ALU has three select bits to decide which of its 8 operations to perform, so the ALU_SEL PLD should have three 1-bit outputs instead of one. This can be done by adding additional outputs to the Output tab in the Combinational Analysis dialog. Name the outputs: S2, S1, and S0.
    • Another note about the ALU_SEL PLD: you can disregard timing signals for this PLD, as the ALU is not a synchronous component. When (in regards to t-values) the ALU does its operations is irrelevant since the ACC will not load the calculated value unless it is signaled to do so. That is, you only need d-values in your expressions for the ALU_SEL PLD.

NOTE: Be sure to include the appropriate clock input (!clock) for all the PLDs involving LOAD signals!

Procedure

  1. Examine microcode for the RSC and look for every reference for the needed signal.
  2. Obtain the logical expression for the needed signal and translate the control signal to a signal that Logisim understands.
  3. Add the Logisim expression to your Control_PLDs.txt file.
  4. Open the PLDs.circ circuit file and and create a part appropriately named for the assigned PLD.
  5. Open Combinational Analysis and use the Expression-analysis sytem to create the circuit. Enter the appropriate input names on the Inputs tab. Enter the appropriate output name on the Outputs tab. The output name should be the same as the circuit name. Don't forget to click ENTER to register the expression to the chosen output.
  6. Select Build Circuit and name the circuit after its output and generate the circuit. Be sure to select the PLDs.circ as the destination.

Requirements

  • All normal conventions apply, with the exception that all input pins on PLDs will face east and all output pins will face west.

Submission

  • A zipped file containing:
    • PLDs.circ
    • Control_PLDs.txt containing ALL of your Boolean expressions up to this point
